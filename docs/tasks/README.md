# Tasks Workflow

Sempre que houver trabalho novo:

1. Criar uma task em `docs/tasks` com objetivo, escopo e status.
2. Atualizar progresso durante a execucao.
3. Marcar como `done` com resultado e proximos passos.

## Convencao de nome

`TASK-XXXX-curto-titulo.md`

Exemplo:
- `TASK-0001-bootstrap-mvp.md`
- `TASK-0002-research-analysis.md`

## Estrutura padrao

- Status: `todo | in_progress | blocked | done`
- Owner
- Objetivo
- Entregaveis
- Checklist
- Riscos
- Resultado
